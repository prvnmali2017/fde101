# Pilot Handoff Checklist

Complete before leaving a customer engagement.

## Documentation

- [ ] Architecture diagram (current state)
- [ ] Deployment runbook (step-by-step)
- [ ] Environment variables / secrets inventory (references only — no values)
- [ ] API documentation (Swagger/OpenAPI or equivalent)
- [ ] Known limitations and workarounds documented
- [ ] FAQ for customer ops team

## Access & Ownership

- [ ] All repos transferred or access granted to customer team
- [ ] Cloud resources tagged with owner and cost center
- [ ] IAM roles documented — who can deploy, who can read logs
- [ ] Secrets rotated if shared credentials were used during pilot

## Operations

- [ ] Monitoring dashboards configured
- [ ] Alerting rules defined with runbook links
- [ ] Backup / data retention policy documented
- [ ] Incident escalation path defined
- [ ] On-call contact for first 2 weeks post-handoff agreed

## Knowledge Transfer

- [ ] 1-hour walkthrough session completed with customer team
- [ ] Demo recording shared
- [ ] "How to add new documents" procedure documented
- [ ] "How to update prompts / model" procedure documented

## Evaluation & Next Steps

- [ ] Pilot metrics summary (before/after if available)
- [ ] Recommended production improvements with effort estimates
- [ ] Customer sign-off on handoff received
